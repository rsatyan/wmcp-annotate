The WebMCP Adoption Toolkit

Open source tooling to make any website AI-agent ready. Built for the W3C standard that's changing how agents interact with the web.

$ npm install -g wmcp-annotate

The Thesis

In February 2026, Chrome shipped WebMCP — a W3C standard that lets websites declare structured tools for AI agents.

This isn't incremental. It's architectural. But 99% of websites have zero WebMCP annotations.

Adoption needs tooling. That's why this exists.

Before WebMCP: Agents scraped DOM (fragile) or parsed screenshots (expensive). Every site needed custom automation.

After WebMCP: Websites declare capabilities. Agents discover and call tools directly. Standard interface, universal compatibility.

What It Does

# Scan — discover actionable elements
wmcp-annotate scan https://example.com

# Suggest — generate tool definitions with AI
wmcp-annotate suggest https://example.com

# Generate — output production code
wmcp-annotate generate https://example.com --format react

# Validate — check spec compliance
wmcp-annotate validate https://example.com

Works with any AI provider — Anthropic, OpenAI, Ollama, or any OpenAI-compatible API. You bring your keys. No lock-in.

Why Open Source?

WebMCP adoption benefits everyone building AI agents. Gatekeeping the tooling slows the ecosystem.

This tool is MIT licensed. Fork it. Ship it. Make it better.

Technical Details

  • Scanning: Playwright-based, handles SPAs and JavaScript rendering
  • AI Integration: Pluggable backends, user-provided API keys
  • Code Generation: JavaScript, TypeScript, React, Vue, Svelte
  • Validation: CI-friendly, detailed compliance reports
